# 주차장 기본정보 보강 전략

`parking_lots` 테이블의 운영시간·요금·면수 결손을 줄여 상세 페이지 품질과 검색 가치를 높이는 필드별 보강 워크플로우.

## When to Activate

- `parking_lots` 보강 스크립트를 작성하거나 수정할 때
- 어떤 소스를 먼저 시도해야 할지 판단이 필요할 때
- 새 보강 파이프라인을 설계하거나 기존 파이프라인을 개선할 때
- 결손 통계를 보고 다음 실행 단위를 결정할 때

결손의 대부분은 `KA-*` 소스에 집중. 공공데이터(`KA/NV` 자동 매칭률 6.35%)만으로는 해결 불가.

## 소스별 역할 정의

| 소스 | 강점 | 약점 | 역할 |
|------|------|------|------|
| 공공데이터 | 운영시간·요금·면수 구조화 품질 높음 | 커버리지 제한, 매칭률 낮음 | 정본. 공영/공공성 주차장 보강 |
| 카카오 상세 | 운영시간, 요금, 무료 여부 | 면수 거의 없음 | `KA-*` 운영시간/요금 1차 보강 |
| 네이버 플레이스 상세 | 운영시간, 전화번호, 휴무 메모 | 파싱 난이도·구조 편차 | 카카오 실패분 2차 보강 |
| 블로그/카페 | 실제 이용 팁, 혼잡도, 할인 조건 | 기본정보 정확도 변동 큼 | `notes`·AI 요약 보조 |

## 필드별 보강 규칙

### 운영시간

**우선순위:** 카카오 상세 → 네이버 플레이스 상세 → 공공데이터 → 블로그

- 시작/종료 시간이 명확할 때만 반영
- 24시간: `00:00 ~ 24:00` 통일
- 블로그 단일 출처는 `notes`만, canonical 필드는 보수적으로

### 요금

**우선순위:** 카카오 상세 → 네이버 플레이스 상세 → 공공데이터 → 블로그

- `base_time`, `base_fee`, `extra_time`, `extra_fee`, `daily_max` 세트로 취급
- 일부 값만 있어도 부분 업데이트 가능, 기존 유효값 덮어쓰기 금지
- 무료 여부는 명시 근거 있을 때만 `is_free = 1`
- 블로그 기반 요금은 교차 근거 없으면 `notes`만

### 면수 (별도 트랙)

**우선순위:** 공공데이터 → 별도 구조화 원천 → 텍스트 추출(보수적)

- 카카오 상세에서 거의 나오지 않으므로 별도 트랙으로 관리
- 블로그 텍스트 추출값은 신뢰도 낮음 — canonical 바로 반영 금지
- 공공데이터 매칭 재정비로 먼저 회수

## 실행 단계 (Phase)

### Phase 1 — 운영시간 복구

```
대상: KA-* + 노외 + 운영시간 누락
방법: scripts/enrich-kakao-place.ts 배치 실행
순서: 샘플 100건 → 500건 단위 확장
```

### Phase 2 — 요금 보강

```
대상: KA-* + 운영시간 채워졌지만 요금 누락
방법: 카카오 상세 요금 추출 → 실패분 네이버 플레이스 2차
```

### Phase 3 — 면수 전용 보강

```
대상: total_spaces <= 0
방법: 공공데이터 매칭 재정비 → 추가 구조화 원천 탐색
주의: 텍스트 추출은 별도 검증 후 반영
```

## 우선순위 정렬

### 현재 (결손량 기준)

1. `KA-*` + `노외` + 운영시간 누락
2. 도시권 우선: 서울 → 경기 → 인천 → 부산 → 대구
3. 결손 많은 순: 경북, 강원, 경남, 전남, 충남

### 향후 (GA 결합 시)

`/wiki/*` 페이지뷰 + landing 유입 + active users + 결손 개수 조합  
→ "트래픽 높은데 기본정보 빈 페이지" 우선

## 스크립트 목록

| 스크립트 | 역할 |
|----------|------|
| `scripts/enrich-from-public-data.ts` | 공공데이터 매칭 보강 |

| `scripts/enrich-from-blog-search.ts` | 블로그 기반 보조 보강 |
| `scripts/enrich-targeted.ts` | 타깃 지정 보강 |

### 개선 필요 사항

1. 카카오 상세 결과에 필드별 성공 통계 출력
2. 네이버 플레이스 상세 전용 스크립트 추가
3. 보강 대상 추출 쿼리 표준화
4. 면수 보강 트랙 분리
5. 결과 리포트 CSV/JSON 자동 생성

## 핵심 원칙

1. **필드별 최강 소스 분리** — 운영시간·요금·면수를 같은 파이프라인으로 보지 않는다
2. **구조화 우선** — 블로그는 보조 근거로만
3. **빈 필드 채우기 ≠ 기존 값 교정** — 두 작업을 분리한다
4. **provenance 보존** — 보강 출처와 반영 시각을 항상 기록한다
5. **보수적 무료 판정** — UI 문구 해석 오류 방지 우선

## 한 줄 요약

```
KA 노외 운영시간 누락 → 카카오 상세 배치 보강
→ 실패분만 네이버/블로그 2차 보강
→ 면수는 별도 프로젝트로 처리
```
