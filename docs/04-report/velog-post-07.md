# [쉬운주차장] 13만 건 크롤링해서 쓸 만한 건 1.2만 건이었다 #7

## 1. 한 달 돌렸더니 13만 건

6화에서 크롤링 파이프라인 만든 얘기를 했었는데, 그걸 한 달 정도 돌린 결과입니다.

| 소스 | 수집량 |
|------|--------|
| 네이버 블로그 | 56,807 |
| DuckDuckGo | 40,288 |
| 네이버 카페 | 35,450 |
| 티스토리 | 527 |
| 유튜브 | 139 |
| **합계** | **133,277** |

네이버 블로그+카페가 9만 건으로 압도적. 한국에서 "주차장 후기" 검색하면 결국 네이버로 수렴합니다. DDG는 네이버에서 안 잡히는 티스토리, 커뮤니티 글을 보완하는 용도.

근데 13만 건을 그대로 쓸 수는 없습니다. 열어보면 부동산 분양 글, 웨딩홀 주차 안내, 월주차 중개 같은 게 절반 가까이 됨.

---

## 2. 절반 가까이가 쓰레기

6화에서 만든 Claude Haiku 기반 AI 필터를 돌렸습니다. 아직 4.7만 건이 큐에서 대기 중이긴 한데, 처리된 8.6만 건 기준으로 보면:

- **통과**: 49,532건 (57.5%)
- **탈락**: 36,631건 (42.5%)

42.5%가 탈락. 거의 절반이 쓸모없는 데이터였음.

탈락 사유가 재밌는데:

| 사유 | 건수 | 비율 |
|------|------|------|
| 주차와 무관 | 17,370 | 47.4% |
| 부동산 분양/매매 | 13,089 | 35.7% |
| 뉴스 기사 | 2,601 | 7.1% |
| 광고 | 2,104 | 5.7% |
| 월주차 중개 | 1,201 | 3.3% |
| 웨딩홀/행사장 | 239 | 0.7% |

**부동산이 35.7%.** "주차장"이란 키워드로 크롤링하면 주차 경험 글보다 "강남역 오피스텔 분양, 주차 100면 확보" 같은 부동산 글이 더 많이 잡힙니다. 이건 예상 못 했음.

가장 많은 "주차와 무관"은 맛집 포스팅에서 "주차장 있어서 편했어요~" 한 줄 언급하는 수준의 글들. 이런 건 주차 난이도 판단에는 의미가 없으니까 버림.

## 3. 필터 통과해도 끝이 아님

AI 필터를 통과한 49,532건이 바로 점수에 들어가는 건 아닙니다. 이 블로그 글이 **우리 DB에 있는 3.5만 개 주차장 중 어디**에 대한 글인지 매칭이 되어야 함.

6화에서 만든 FTS + AI 하이브리드 매칭을 돌린 결과:

```
크롤링 원본:  133,277건
  ↓ AI 필터
유효 콘텐츠:   49,532건
  ↓ 주차장 매칭
최종 연결:    12,551건
```

13만에서 **1.2만**. 9.4%만 살아남았습니다.

소스별로 보면 매칭률 차이가 꽤 큼.

| 소스 | 매칭 성공 | 원본 대비 |
|------|-----------|-----------|
| 네이버 블로그 | 8,131 | 14.3% |
| 네이버 카페 | 2,399 | 6.8% |
| DuckDuckGo | 1,407 | 3.5% |
| 티스토리 | 600 | - |

네이버 블로그가 매칭률이 가장 높습니다. 이유는 단순한데, 블로거들이 제목에 주차장 이름을 잘 써줌. "강남 타임스퀘어 주차 후기" 같은 제목이면 FTS에서 바로 후보가 잡히고, 신뢰도 high로 AI 확인 없이 바로 연결됨.

반면 DDG는 3.5%로 낮음. DDG 경유로 들어오는 건 커뮤니티 글이나 개인 블로그가 많은데, 제목이 "여기 주차 개힘듦 ㅋㅋ" 같은 식이라 어떤 주차장인지 특정이 안 됨.

## 4. 3.5만 개 중 5,634개에 데이터가 붙었다

최종 결과. **34,719개 주차장 중 5,634개(16.2%)** 에 웹 데이터가 연결됐습니다.

16%면 적어 보이는데, 이 5,634개가 사실 사람들이 실제로 가는 주차장임. 블로그에 후기가 올라올 정도면 이용량이 있다는 뜻이니까요. 나머지 84%는 동네 소규모 공영주차장이나 아파트 단지 주차장 같은 곳들이라 애초에 블로그 글이 안 나옴.

주차장당 평균 웹 소스 수는 2.2건. 많은 곳은 수십 건이 연결되어 있고, 이게 베이지안 스코어링에 반영되면서 사용자 리뷰 없이도 난이도 추정이 가능해졌습니다. "리뷰 없음"이던 주차장에 "추정 3.2 (블로그 기반)" 같은 점수가 붙기 시작함.

## 5. Cron으로 자동화해놓으니 편하다

이 파이프라인 전체를 Cloudflare Workers Cron Trigger에 올려놨습니다.

```
매시 정각  → 네이버/유튜브 크롤링 → AI 필터 → 매칭 → 스코어링
매시 30분  → DuckDuckGo 크롤링
```

DDG를 30분에 따로 돌리는 이유는 6화에서 말한 Workers subrequest 한도 때문. `wrangler.jsonc`에 cron 두 줄 추가한 게 전부입니다.

```jsonc
{
  "triggers": {
    "crons": ["0 */1 * * *", "30 */1 * * *"]
  }
}
```

오늘 기준 실행 상태:

| 크롤러 | 마지막 실행 | 누적 처리 |
|--------|-----------|----------|
| 네이버 블로그 | 오늘 14:01 | 14,460건 |
| DDG 검색 | 오늘 14:32 | 8,205건 |
| 스코어링 | 오늘 14:01 | 15,161건 |

한 번 세팅해놓으니까 손 안 대도 매시간 알아서 돌아감. 새 블로그 글이 올라오면 수집하고, 필터링하고, 주차장에 매칭하고, 점수 갱신까지.

Workers Free plan에서 이게 되는 이유는, 매번 전체를 풀스캔하는 게 아니라 증분 처리를 하기 때문.

- 크롤링: `INSERT OR IGNORE`로 중복 스킵
- AI 필터: `ai_filtered_at IS NULL`인 것만 처리
- 매칭: `filter_passed = 1 AND matched_at IS NULL`인 것만
- 스코어링: 마지막 실행 이후 매칭된 주차장만 재계산

각 단계가 "아직 안 한 것만" 골라서 처리하니까 30초 wall time 안에서 충분히 돌아갑니다.

## 6. 정리

```
[크롤링]     133,277건 수집
                │
[AI 필터]    86,163건 처리 → 36,631건 탈락 (42.5%)
                │
[유효]       49,532건 통과
                │
[매칭]       12,551건 주차장 연결
                │
[커버리지]   5,634개 주차장 / 34,719개 전체 (16.2%)
```

13만에서 1.2만. 90%가 버려졌는데, 남은 10%가 진짜 쓸 만한 주차 경험 데이터임. 이게 cron으로 매시간 자동으로 쌓이고 있으니까, 시간이 지날수록 커버리지는 올라갈 겁니다.

사이드 프로젝트에서 크롤링 파이프라인 만들고 자동화까지 해본 건 처음인데, 한 번 세팅해놓으면 진짜 편함. 데이터가 알아서 쌓이는 걸 지켜보는 재미가 있습니다.
