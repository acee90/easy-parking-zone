# 파킹이지(Parking Easy) 마케팅 페르소나 가이드

이 문서는 서비스의 대외 소통, 커뮤니티 바이럴, 콘텐츠 제작 시 일관성을 유지하기 위한 전략적 기준입니다.

## 🐣 핵심 페르소나: "서울 사는 2개월 차 긁힌 초보 개발자"

### 1. 프로필 (Identity)
- **거주지:** 서울 (좁은 골목과 복잡한 주차장의 전쟁터)
- **운전 경력:** 이제 막 2개월 된 병아리 운전자.
- **결정적 사건(트라우마):** 
    - 강남에서 쏘카 반납하다가 좁은 통로에서 후진 중 휠 긁어먹은 기억.
    - 스타벅스 갔다가 만차라서 당황하며 식은땀 흘린 기억.
- **개발 스타일:** '바이브 코딩' (Claude, Gemini 등 AI를 적극 활용하여 빠르게 구현).
- **기술 스택:** Naver Maps API + React(TanStack Query) + Cloudflare D1/Workers (가볍고 실속 있는 현대적 스택).

### 2. 브랜드 미션 (Mission)
- "주행보다 주차가 더 무서운 초보들을 위한 '안심 주차 가이드' 제공"
- "유튜브 '10시10분' 같은 헬주차장을 미리 피할 수 있게 돕기"

### 3. 소통 원칙 (Tone & Manner)
- **담백함:** 화려한 미사여구 배제. "벌써 한 번 해먹음...ㅠㅠ" 같은 솔직한 팩트와 이모지 사용.
- **공감 우선:** 기술적 자랑보다 "주차장 입구의 압박감" 같은 초보의 실전 고통에 먼저 공감.
- **낮은 자세:** "내가 다 만들었다"가 아니라 "형님들의 집단지성이 필요하다"는 태도로 데이터 제보 유도.
- **투명한 공유:** AI 도움을 받는 과정(바이브 코딩)이나 개발 삽질기를 가감 없이 공유 (Build in Public).

### 4. 채널별 전략 (Channel Strategy)
- **Velog:** 개발 삽질기와 기획 의도 상세 공유 (기술적 신뢰도 및 팬덤 확보).
- **커뮤니티 (클리앙/블라인드/동호회):** 도움 요청 및 공감 유도형 게시글 (실질적인 바이럴 및 데이터 확보).
- **숏폼 (릴스/틱톡):** '10시10분' 류의 공포 영상 + 해결책 제시 (대중적 UA).
