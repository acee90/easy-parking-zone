# [쉬운주차장] 마케팅 페르소나 가이드

이 문서는 서비스의 대외 소통, 커뮤니티 바이럴, 콘텐츠 제작 시 일관성을 유지하기 위한 전략적 기준입니다.

## 🐣 핵심 페르소나: "주차장 DB는 만들었는데 리뷰가 없는 개발자"

### 1. 프로필 (Identity)
- **정체성:** 서울 사는 운전 2~3개월 차 초보. 주차하다 차 긁은 경험이 있는 실용주의 개발자.
- **개발 스타일:** AI(Claude/Gemini)를 도구로 적극 활용하되, 데이터의 정합성과 인프라 효율성을 고민함.
- **기술 스택:** Naver Maps API + React + Cloudflare D1/Workers (서버리스 환경 선호).

### 2. 소통 원칙 (Tone & Manner)
- **담백하고 실무적:** 과장된 표현을 지양하고 "데이터 노이즈", "정합성 문제", "인프라 번거로움" 등 개발 현장의 용어 사용.
- **문제 해결 중심:** 감정적인 호소보다 "데이터가 없어서 이렇게 해결해 봤다"는 기술적/논리적 과정 공유.
- **공감과 제보 유도:** "나도 주차가 무섭다"는 공감대를 바탕으로, 집단지성(데이터 제보)을 정중히 요청.

### 3. 주요 키워드
- **긍정:** 데이터 정제, 베이지안 평균, 인프라 최적화, 신뢰도, 효율성.
- **지양:** 과도한 이모지 사용, 자극적인 공포 묘사, 비속어나 너무 가벼운 유머.
