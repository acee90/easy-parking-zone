# AI Summary 생성 프롬프트

`web_sources.ai_summary` 생성에 사용하는 시스템 프롬프트. 변경 시 `/tmp/build-all-prompts.py`의 `SYSTEM_HEADER`도 함께 업데이트하고 프롬프트 파일을 재생성해야 합니다.

## SYSTEM_HEADER

```
주차장 블로그 본문에서 ai_summary를 생성하세요.

## 좋은 예시
입력 본문: "홍대 공영주차장 방문후기... 기본 30분 1,000원, 이후 10분당 500원, 1일 최대 10,000원... 주말 오후 2시면 이미 만차... 진입로가 좁아 대형차 진입 어려움..."
출력: {"id": 99, "summary": "기본 30분 1,000원, 이후 10분당 500원이며 1일 최대 10,000원입니다. 주말 오후 2시면 만차가 될 정도로 혼잡하며, 진입로가 좁아 대형차 진입이 어렵습니다."}

입력 본문: "용머리해안·산방산 여행... 주차는 산방산 공영주차장에 하시면 됩니다..."
출력: {"id": 99, "summary": "용머리해안·산방산 방문 시 산방산 공영주차장을 추천합니다."}

입력 본문: "꽃길만걷개 애견카페 방문기... 주차는 통일아시아드공원 노상 공영주차장 이용..."
출력: {"id": 99, "summary": "꽃길만걷개 방문 시 통일아시아드공원 노상공영주차장을 추천합니다."}

나쁜 예시 (이렇게 하지 마세요):
- "저렴한 요금이 특징입니다" (수치 없음, 추상적 — 도움 안 됨)
- "주차 가능합니다" (정보 없음)
- "넓고 쾌적한 주차장입니다" (본문 근거 없음, 할루시네이션)
- "홍대 맛집 골목 근처에 위치하며 쇼핑하기 좋습니다" (주차장 정보 아님)
- "공영주차장을 이용할 수 있습니다" (어디 방문 시인지 맥락 없음 — 도움 안 됨)

## 규칙
- 완전한 문장, 경어체(~습니다/~입니다) 필수, 마침표 필수
- 명사 뒤에 "~습니다" 금지 (예: "무료습니다" → "무료입니다")
- 본문에 실제 언급된 수치(요금·시간·거리·주차면수)는 그대로 인용
- 요금 외 언급된 정보(혼잡도·진입 특이사항·주차면 특징)도 포함
- 블로그가 특정 맛집·관광지·시장 방문기인 경우: 방문 목적지 + 추천 주차장을 함께 추출하여 "[목적지] 방문 시 [주차장명]을 추천합니다." 형태로 요약 (요금 정보 없어도 OK)
- 본문에 없는 수치·사실 절대 출력 금지 (할루시네이션 금지)
- 주차장과 무관한 내용(관광지 설명·맛집·날씨 등) 출력 금지
- 이모지, 마크다운, 메타 표현(AI가 분석 등) 금지

## summary='' 조건
- 주차장 이름조차 언급되지 않고 "주차 편리합니다" 같은 막연한 표현만 있음
- 광고·홍보 글
- 방문 목적지가 불분명하여 맥락 파악 불가

## 출력 형식 (JSON array만, 다른 텍스트 금지)
[{"id": 123, "summary": "..."}, ...]

## 입력 데이터
```

## 프롬프트 재생성

```bash
# 청크 디렉토리 기준 전체 재생성
python3 /tmp/build-all-prompts.py

# 특정 범위만 헤더 교체 (prompts-20 디렉토리)
python3 << 'EOF'
import os
NEW_HEADER = "..."  # 위 SYSTEM_HEADER 내용
prompt_dir = "/tmp/ai-summary-prompts-20"
for i in range(0, 207):
    fname = f"{prompt_dir}/prompt-{i:04d}.txt"
    content = open(fname).read()
    idx = content.find("\n### [")
    if idx != -1:
        open(fname, "w").write(NEW_HEADER + content[idx:])
EOF
```
