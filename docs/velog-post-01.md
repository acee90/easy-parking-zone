# [Parking Easy] "주차가 무서워서 경기도만 가요" 전국 주차 난이도 지도 개발기 #1

## 1. 운전보다 주차가 더 무서운 이유

안녕하세요. 운전면허 딴 지 3개월 차, 서울 시내 주행보다 주차가 더 무서운 초보 개발자입니다.

차선 변경이나 신호 대기도 여전히 긴장되지만, 저에게 가장 큰 스트레스는 주차장입니다. 유튜브 ‘10시10분’ 채널에 나오는 좁고 경사진 주차장들을 보며 매번 남 일 같지 않다고 느꼈습니다. 결국 요즘은 주차 걱정 없는 경기도 외곽으로만 드라이브를 다니고 있는 실정입니다.

---

## 2. 몇 번의 아픈 기억 (프로젝트의 시작)

주차 때문에 일상이 불편해지는 경험이 반복되다 보니, 직접 해결해보고 싶다는 생각이 들었습니다.

*   스타벅스 주차장이 만차라 골목길을 수십 번 헤맸던 기억
*   강남에서 쏘카 반납 중 주차장 입구가 너무 좁아 후진하다 차를 긁었던 트라우마

네이버 지도 같은 기존 서비스들은 주차장 위치는 잘 알려주지만, **"초보 운전자가 안심하고 갈 수 있을 만큼 넉넉한가?"**에 대한 답은 주지 않았습니다. 그래서 직접 '난이도'를 시각화한 지도를 만들어보기로 했습니다.

---

## 3. 프로젝트 구상: 쉬운 주차장 (가제)

제가 구상 중인 기능은 크게 세 가지입니다.

*   **난이도 컬러 코딩:** 초보자도 넉넉히 주차 가능한 곳은 초록색, 좁거나 복잡한 헬 주차장은 빨간색으로 구분
*   **데이터 소스:** 네이버 지도 API와 연동하고, 실제 사용자 경험(입구 좁음, 경사 심함 등)을 태그로 수집
*   **추가 정보:** 가능하면 입구 사진과 주차 구획 크기, 실시간 빈자리 정보까지 제공

---

## 4. 기술 스택: 프론트엔드부터 인프라까지

초기 단계라 가볍고 효율적인 도구들을 선택했습니다.

*   **Frontend:** React + TanStack Query (상태 관리 최강자)
*   **Map API:** Naver Maps API
*   **Backend:** Cloudflare Workers + D1 (서버리스로 가볍고 빠르게 구성)

---

## 5. 마무리하며

현재 데이터를 어떻게 효율적으로 수집할지 고민 중입니다. 저처럼 "주차 때문에 목적지를 포기하는" 분들이 없도록 유의미한 지도를 만들어보고 싶습니다.

다음 포스트에서는 이번에 작업한 초기 프로토타입 화면과, 유튜브 데이터를 활용한 수집 과정을 공유하겠습니다.

끝.
