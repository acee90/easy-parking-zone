# AI-generated Content QA Standard

## 목적

주차장 상세 페이지용 자체 콘텐츠는 검색엔진용 텍스트를 늘리는 것이 아니라, 사용자가 방문 판단에 참고할 수 있는 근거 기반 정보를 제공해야 한다. 이 문서는 생성된 `aiSummary`, `aiTipPricing`, `aiTipVisit`, `aiTipAlternative`를 같은 기준으로 검수하고, 수정/재생성 여부를 결정하기 위한 상시 운영 기준이다.

## 기본 원칙

1. 공식/구조화 데이터와 충돌하지 않는다.
2. 근거가 없는 요금, 무료 여부, 혼잡도, 진입 난이도를 단정하지 않는다.
3. 주차장별 고유 정보가 없는 일반론은 통과시키지 않는다.
4. 여러 페이지에 같은 템플릿 문장이 반복되면 재생성한다.
5. `hell/easy` 큐레이션 문구와 SEO 요약/팁의 역할을 섞지 않는다.

## 문체 기준

허용:

- 중립적이고 실용적인 안내 문체
- “확인해 두면 좋습니다”, “비교해 볼 만합니다”처럼 행동을 돕는 표현
- 근거가 약한 항목에는 “방문 전 확인 권장”, “제공된 정보 기준” 같은 제한 표현
- 주차장명, 지역, 유형, 운영/요금 데이터, 리뷰/웹소스에서 확인되는 특징을 반영한 문장

금지:

- 광고성 문체: “최고의”, “완벽한”, “강력 추천”
- 과장 단정: “항상 여유롭다”, “무조건 저렴하다”, “100% 무료”
- 근거 없는 혼잡 예측: “주말에도 넉넉하다”, “피크타임에도 문제없다”
- 출처 없는 비교: “주변에서 가장 싸다”, “근처 최고 가성비”
- 빈 템플릿: “방문 목적에 맞게 이용하면 좋습니다”처럼 어느 주차장에도 붙일 수 있는 문장

## 필드별 검수 기준

| 필드 | 통과 기준 | 실패 기준 |
|---|---|---|
| `aiSummary` | 해당 주차장의 위치/유형/데이터/웹소스 특징이 최소 2개 이상 반영됨 | 모든 주차장에 재사용 가능한 일반 설명 |
| `aiTipPricing` | 요금/무료/정기권/할인 표현이 DB 또는 명시 근거와 일치함 | 유료 주차장을 무료처럼 표현하거나 금액을 임의 생성 |
| `aiTipVisit` | 진입, 운영시간, 목적지 접근성, 방문 전 확인 포인트 등 실제 행동에 도움 됨 | 혼잡도/난이도를 근거 없이 단정 |
| `aiTipAlternative` | 대안 탐색, 인근 비교, 만차 대비처럼 보수적 표현 사용 | 특정 대안이 더 낮다고 근거 없이 단정 |

## 공식 데이터 충돌 금지 항목

아래 항목은 공식 데이터나 구조화 필드가 우선이다. 생성 문구가 충돌하면 실패 처리한다.

- `is_free=false` 또는 요금 필드가 있는데 “무료”, “완전 무료”라고 표현
- `is_free=true`인데 “요금 부담”, “비싼 편”이라고 표현
- 운영시간이 비어 있는데 “24시간 운영”이라고 표현
- 운영 종료 시간이 있는데 “언제든 이용 가능”이라고 표현
- 총면수 정보가 없는데 “대형 주차장”, “넓은 주차장”이라고 단정
- 공영/민영/부설 유형을 잘못 바꿔 표현
- 주소/행정구역이 다른 목적지나 다른 주차장과 섞임

## 금지 표현 목록

자동 검사 또는 수동 검수에서 발견되면 문맥을 확인하고, 근거가 없으면 실패 처리한다.

- 무조건
- 반드시 비어 있음
- 100%
- 완전 무료
- 항상 여유
- 절대
- 최고의
- 가장 저렴
- 가장 가까운
- 확실히
- 걱정 없이
- 주말에도 넉넉
- 피크타임에도 문제없음

## 검수 절차

1. 생성 대상 샘플을 무작위가 아니라 우선순위 상위, 데이터 풍부, 데이터 빈약, 공영, 민영/부설이 섞이도록 추출한다.
2. 각 주차장마다 `aiSummary`와 팁 3종을 모두 검수한다.
3. 항목별로 `PASS`, `FIX`, `REGEN`, `DROP` 중 하나를 기록한다.
4. `FIX`는 단순 표현 수정으로 해결 가능한 경우에만 사용한다.
5. `REGEN`은 근거는 충분하지만 문체/구성/반복 문제가 큰 경우에 사용한다.
6. `DROP`은 근거 부족, 오매칭, 공식 데이터 충돌처럼 생성 대상에서 제외해야 하는 경우에 사용한다.

## 판정 기준

| 판정 | 의미 | 후속 조치 |
|---|---|---|
| `PASS` | 그대로 노출 가능 | 저장/배포 가능 |
| `FIX` | 일부 표현만 수정하면 사용 가능 | 사람이 수정하거나 deterministic rewrite |
| `REGEN` | 근거는 있으나 생성 품질이 낮음 | 프롬프트 보강 후 재생성 |
| `DROP` | 근거 부족 또는 데이터 충돌 | 해당 주차장 생성 결과 미노출 |

운영 게이트 (Batch 단위):

- `PASS + FIX` 비율이 80% 이상이면 다음 단계 검토 가능
- 공식 데이터 충돌률이 5%를 넘으면 프롬프트/검증 로직 수정 전 확대 금지
- `DROP` 비율이 20%를 넘으면 대상 선정 기준을 다시 조정
- 동일/유사 템플릿 반복이 10건 이상 보이면 재생성

## 검수 체크리스트

각 샘플 row에 대해 아래 항목을 확인한다.

- 주차장명/주소/지역이 다른 장소와 섞이지 않았는가
- 요금/무료 여부가 구조화 데이터와 충돌하지 않는가
- 운영시간 표현이 확인 가능한 값보다 강하게 단정되지 않았는가
- 총면수나 규모를 근거 없이 표현하지 않았는가
- 혼잡도, 진입 난이도, 주말 여유 여부를 근거 없이 예측하지 않았는가
- 웹소스가 광고/무관/오매칭일 때 그 내용을 요약하지 않았는가
- 같은 문장 구조가 여러 주차장에 반복되지 않았는가
- 사용자가 실제로 방문 전 확인하거나 비교할 수 있는 정보가 있는가
- `hell/easy` 큐레이션 메시지와 의미가 충돌하지 않는가
- 문장이 너무 짧거나 비어 보이지 않는가

## 수정/재생성 기준

단순 수정으로 충분한 경우:

- 표현이 과장되어 있지만 근거 자체는 맞음
- “가장”, “무조건” 같은 단어만 제거하면 자연스러움
- 주차장 유형이나 행정구역 표기가 사소하게 어색함

재생성이 필요한 경우:

- 전체 문장이 일반론으로 구성됨
- 같은 템플릿이 반복됨
- 팁 3종의 역할이 서로 겹침
- 웹소스 특징을 잘못 해석했지만 올바른 근거가 입력에 있음

미노출해야 하는 경우:

- 공식 데이터와 핵심 정보가 충돌함
- 다른 장소/다른 주차장 정보를 섞음
- 근거가 거의 없어 추측만 남음
- 오매칭 웹소스나 광고성 글을 주요 근거로 사용함

## QA 기록 포맷

검수 결과는 아래 컬럼으로 기록한다.

| 컬럼 | 설명 |
|---|---|
| `lot_id` | 주차장 ID |
| `lot_name` | 주차장명 |
| `source_density` | high / medium / low |
| `summary_result` | PASS / FIX / REGEN / DROP |
| `pricing_result` | PASS / FIX / REGEN / DROP |
| `visit_result` | PASS / FIX / REGEN / DROP |
| `alternative_result` | PASS / FIX / REGEN / DROP |
| `failure_reason` | 실패 또는 수정 사유 |
| `action` | keep / edit / regenerate / hide |
